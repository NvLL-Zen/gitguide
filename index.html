GitGuide: A simple git command cheatsheet

Authenticating git with github

There are several methods to authenticate to github, one that i prefer to use is github CLI, to get started on this. Head over to the official github repo and follow the instructions to install it in your operating system.

Once installed, you should be able to enter the command "gh" in your command line and see something similar to this:

– □ x 

~> gh

work seamlessly with GitHub from the command line.       

USAGE

gh (command) (subcommand) [flags] .......

If your output is similar to that, congratulations. Github CLI is installed

Next step is to authenticate git with your github, enter the command "gh auth login" in your command line

You will be prompted some questions, do the following answer stated below:

– □ x 

~> gh auth login

? What account do you want to log into? Github.com

? What is your preferred protocol for Git operations on this host? HTTPS

? How would you like to authenticate GitHub CLI? Login with a web browser       

! First copy your one-time code: XXXX-XXXX

Press Enter to open github.com in your browser...

Now press enter and go to your web browser

Paste or enter the code given, and authorize the github

Once it's done, head over to the terminal and you should see this

– □ x 

....

✓ Authentication complete.

- gh config set -h github.com git_protocol https       

✓ Configured git protocol

! Authentication credentials saved in plain text

✓ Logged in as (Your github username)

If your output is something like that, then the git is authorized with github

Next is just configuring your git config's global username and email

To do that follow the step below

– □ x 

~> git config --global user.email "(Your github email)"       

~> git config --global user.name "(Your github username)"

If you did that correctly and there is no error exception, git has been configured on your computer

Git basic commands

Creating a repo or a repository

First head to your project folder you want to upload to git

Then open your terminal in that and type in this command

– □ x 

~> git init      

It should create a git repo within the project folder

Adding and commiting a change

To add files to your repo, simply type this command

– □ x 

~> git add (filename)      

Or

– □ x 

~> git add .      

This will add files according to the syntax your using, the '.' means all files in that project folder

Now that it's added you can now commit or save your changes

To do that, type and enter this command:

– □ x 

~> git commit -m "(Your message)"      

You can change "Your message" to something like "Fixed the bug at line *" or something useful

Creating and pushing git repo to github

Next if you want to save that to the git repo you would have to push it

But before that, you have to create the github repo first

To do that, simply type and enter this command

– □ x 

~> git repo create      

After that you'll be prompted some questions, answer as i do

– □ x 

~> ......      

What would you like to do? Push an existing local repository to GitHub       

? Path to local repository .

? Repository name (repository-name)

? Description (repository-description)

? Visibility Public

✓ Created repository (github-username/repository-name) on GitHub

? Add a remote? Yes

? What should the new remote be called? origin

✓ Added remote (https://github.com/github-username/repository-name.git)

? Would you like to push commits from the current branch to "origin"? Yes

After that, you should see the push process in the terminal

Now with that out of the way you could now use git in the project folder without that hussle

Pushing git commit to github (after creating the github repo)

After creating the github repo and push the existing commit to github, you can now use the push command

To do so, make sure you commited the changes and just type and enter this command

– □ x 

~> git push      

You should see the push process as in the push process you made when you create the github repo above

Congrats, you just learned the basic of using git as a VCS(Version Control System)
